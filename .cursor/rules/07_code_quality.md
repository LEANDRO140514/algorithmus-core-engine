# Calidad de código

## Estándares

* TypeScript strict
* tipos explícitos
* evitar any
* funciones pequeñas y puras

---

## Errores

* usar clases de error tipadas
* no usar throw genérico sin contexto

---

## Logs

* no usar console.log
* usar logger estructurado

---

## Naming

* carpetas: snake_case
* archivos: kebab-case
* variables: camelCase

---

## Imports

* evitar dependencias circulares
* usar paths claros
